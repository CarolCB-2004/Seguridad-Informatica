    • Objetivo de la Práctica

    El objetivo principal de esta práctica es diseñar e implementar una VPN (Red Privada Virtual) de sitio a sitio (Site-to-Site) empleando el protocolo IPSec sobre routers Cisco 1941.

    El fin es establecer un túnel seguro y cifrado que permita la comunicación transparente y confidencial entre dos redes locales (192.168.1.0/24 y 192.168.3.0/24) a través de una red pública insegura (simulada por un router ISP). Se busca garantizar la integridad y confidencialidad de los datos, replicando un escenario corporativo real.

    Topología implementada

    El enrutador del proveedor actuó como puente para emular la conexión a la red pública. La infraestructura consta de:

    • Equipos de red: R1 (Sede Principal), R3 (Sede Secundaria) e ISP (Proveedor).
    • Switches: Para el tendido de las redes internas.
    • Terminales: PC-A y PC-B para comprobar el alcance total.

    Desarrollo

    Configuración Primaria y Protocolos de Enrutamiento

    Se asignaron direcciones IP a las interfaces GigabitEthernet y rutas estáticas por defecto.

    Configuración de R1
    interface g0/1
 ip address 192.168.1.1 255.255.255.0
 no shutdown
interface g0/0
 ip address 209.165.100.1 255.255.255.0
 no shutdown
ip route 0.0.0.0 0.0.0.0 209.165.100.2
    Configuración del ISP
    interface g0/0
 ip address 209.165.100.2 255.255.255.0
 no shutdown
interface g0/1
 ip address 209.165.200.2 255.255.255.0
 no shutdown
    Configuración de R3
    interface g0/1
 ip address 192.168.3.1 255.255.255.0
 no shutdown
interface g0/0
 ip address 209.165.200.1 255.255.255.0
 no shutdown
ip route 0.0.0.0 0.0.0.0 209.165.200.2

    Verificación de Conectividad Básica

    Antes de configurar la VPN, se validó el alcance entre los routers de las sedes:

    R1#ping 209.165.200.1
Type escape sequence to abort.
Sending 5, 100-byte ICMP Echos to 209.165.200.1, timeout is 2 seconds:
Success rate is 100 percent (5/5), round-trip min/avg/max = 0/0/0 ms

    Configuración de IPSec

    Política ISAKMP (Fase 1)

    Definición de parámetros para el canal seguro:

    R1(config)#crypto isakmp policy 10
R1(config-isakmp)#encryption aes 256
R1(config-isakmp)#authentication pre-share
R1(config-isakmp)#group 5
R1(config-isakmp)#exit
    Verificación del túnel: Se confirmó que el tráfico entre las redes 192.168.1.0 y 192.168.3.0 se encapsula y cifra correctamente.

    Conclusión

    La implementación permitió comprender el proceso de establecer comunicaciones seguras mediante dos fases: Fase 1 (ISAKMP) para autenticación y Fase 2 (IPSec) para el cifrado mediante transform sets.

    Se resolvieron desafíos técnicos como la activación de licencias securityk9 y la gestión de crypto maps. El éxito de la práctica se validó mediante la observación de paquetes ESP cifrados, garantizando que la información es privada durante su tránsito por redes públicas.
